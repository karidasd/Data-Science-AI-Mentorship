# 💼 ATS-Friendly Resumes & Portfolio Strategy

When you apply for a job, a human doesn't read your resume first. An **ATS (Applicant Tracking System)** does. If your resume isn't formatted correctly, it goes straight to the trash.

## ❌ Top 3 Resume Mistakes
1. **Multiple Columns & Crazy Designs:** ATS software reads left-to-right, top-to-bottom. If you use a fancy Canva template with 2 columns and progress bars for your skills, the ATS will scramble the text.
2. **Missing Keywords:** If the job description asks for `PySpark` and you write `Spark (Python)`, the ATS might miss it. Mirror the job description exactly!
3. **Vague Bullet Points:** "Built a machine learning model." -> This tells the recruiter nothing.

---

## ✅ The "XYZ" Formula for Bullet Points
Google famously recommends the XYZ formula for resumes:
**"Accomplished [X] as measured by [Y], by doing [Z]."**

**Bad:** *Optimized a database.*
**Good:** *Reduced SQL query execution time by 40% (X) resulting in saving $2,000/month in AWS costs (Y), by implementing materialized views and B-Tree indexing (Z).*

### Top 10 Action Verbs to Start Your Bullets:
*Spearheaded, Architected, Engineered, Optimized, Deployed, Orchestrated, Quantified, Formulated, Automated, Streamlined.*

---

## 🎯 Tailoring the SAME Project for Different Roles

If you built an End-to-End project predicting housing prices, you must change the bullet points depending on what role you are applying for!

### Applying for Data Analyst:
- *Spearheaded EDA using Pandas and Seaborn to identify 3 key drivers of real estate pricing.*
- *Architected a Tableau dashboard visualizing neighborhood trends, reducing reporting time by 15 hours/week.*

### Applying for Machine Learning Engineer (MLE):
- *Engineered an XGBoost predictive model, improving baseline RMSE by 22%.*
- *Containerized the model using Docker and orchestrated deployment via GitHub Actions CI/CD to AWS EC2.*

## 📁 The "Killer" Portfolio Strategy

Your GitHub is your proof of work. But recruiters spend exactly 15 seconds looking at it.

**How to make your GitHub look Senior:**
1. **The README is King:** A repo with great code but no README is useless. Your README must have: A Goal, an Architecture Diagram (Mermaid.js), and Instructions on how to run the code.
2. **Pinned Repositories:** Pin your top 4 projects. Do not pin "Homework 1" from your university. Pin "End-to-End Real-Time Kafka Pipeline".
3. **Green Squares:** Try to commit consistently. A highly active contribution graph shows passion.
